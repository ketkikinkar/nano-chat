\subsection{Why the Initial Loss Equals $\ln(50257)$}

At initialization all weights $\sim\mathcal{N}(0, 0.02)$, so logits $\approx 0$. Softmax of all-zero logits is
uniform over 50257 tokens. Cross-entropy of a uniform distribution:
$-\log(1/50257) = \ln(50257) \approx 10.82$. Our measured value is 10.8294 -- gap $<0.01$ confirms
correct pipeline.

\subsection{SFT Loss Function}

\begin{lstlisting}[caption={Complete SFT loss (sft/trainer.py)}]
def sft_loss(model, input_ids, loss_mask):
    B, T  = input_ids.shape
    logits  = model(input_ids)                     # (B, T, vocab_size)
    logits  = logits[:, :-1, :].contiguous()       # drop last (no target)
    targets = input_ids[:, 1:].contiguous()        # shift: target[t] = input[t+1]
    mask    = loss_mask[:, 1:].contiguous()        # align mask with shifted targets

    per_token = F.cross_entropy(logits.view(-1, vocab_size),
                                targets.view(-1), reduction="none").view(B, T-1)
    denom = mask.sum()
    if denom == 0:
        return torch.tensor(0.0, requires_grad=True)
    return (per_token * mask).sum() / denom        # average over ASSISTANT tokens only
\end{lstlisting}

\subsection{Key RL Equations and Intuition}

\begin{center}
\small
\begin{tabular}{@{}L{5.2cm}L{9.0cm}@{}}
\toprule
\TH{Formula} & \TH{Intuition} \\
\midrule
$\hat{A}_i = (r_i - \bar{r}) / (\sigma_r + 10^{-8})$ &
  Group-relative: "this was 2 std above group mean." Scale-invariant.
  $\epsilon$ prevents div-by-zero when all rewards are identical. \\
\TA $\mathcal{L}_\text{PG} = -\hat{A}_i \cdot \log\pi_\theta(y_i|x)$ &
  Negative sign: PyTorch minimizes, we want to maximize reward.
  High-$\hat{A}$ completions get increased log-prob. \\
$\mathcal{L}_\text{KL} = \log\pi_\theta(y) - \log\pi_\text{SFT}(y)$ &
  Divergence from frozen SFT ref. Prevents reward hacking (gaming rules
  with degenerate text). KL coeff 0.1 = 1 nat of KL costs 0.1 loss. \\
\TA $\mathcal{L} = \mathcal{L}_\text{PG} + 0.1 \cdot \mathcal{L}_\text{KL}$ &
  Final loss. Tradeoff: $\alpha{\to}0$ = pure RL, hacking.
  $\alpha{\to}\infty$ = frozen model, no learning. \\
\bottomrule
\end{tabular}
\end{center}

\section{Interview Questions --- Standard Level}
%=============================================================================

\begin{multicols}{2}
\footnotesize

\begin{interviewbox}{Q1: Why scale attention by $\sqrt{d_k}$?}
Dot products have std $= \sqrt{d_k}$.
With $d_k=64$, std$=8$: softmax is nearly one-hot, gradients vanish.
Dividing by $\sqrt{64}=8$ restores std$\approx1$, keeps all heads trainable.
\end{interviewbox}

\begin{interviewbox}{Q2: What is weight tying and what does it save?}
\texttt{lm\_head.weight = wte.weight} shares the same tensor.
Same job, opposite directions. Saves $50257\times768 \approx 38.6$M params (31\% of GPT-2).
Test with \texttt{is}, not \texttt{==}.
\end{interviewbox}

\begin{interviewbox}{Q3: Why must SFT use mask.sum() not seq\_len?}
Only assistant tokens contribute to numerator.
Using \texttt{seq\_len}: 500-token prompt + 10-token answer = 51x dilution.
\texttt{mask.sum()} = 10 = correct denominator. Loss is always average over assistant tokens only.
\end{interviewbox}

\begin{interviewbox}{Q4: How does KV-cache work?}
Keys+values for prior tokens don't change between steps.
Compute once (prefill), cache, reuse.
Each new step processes ONE token: O($T$) not O($T^2$).
Tradeoff: $O(T\cdot L\cdot H\cdot d_h)$ memory grows with context.
\end{interviewbox}

\begin{interviewbox}{Q5: What is group-relative advantage?}
Normalize within G completions: $\hat{A}_i=(r_i-\bar{r})/(\sigma_r+\epsilon)$.
"Reward 0.7 when mean=0.3, std=0.2" becomes $+2.0\sigma$ signal.
Scale-invariant across prompts.
$\epsilon=10^{-8}$ prevents div-by-zero.
\end{interviewbox}

\begin{interviewbox}{Q6: Why is a KL penalty needed in RL?}
Without it: reward hacking.
Model finds degenerate completions that score high but are nonsense
(e.g., "!!!.!.!." for punctuation reward).
KL anchors to frozen SFT ref.
$\alpha=0.1$: 1 nat of KL costs 0.1 in loss.
\end{interviewbox}

\begin{interviewbox}{Q7: Pre-LN vs Post-LN?}
Pre-LN: $x \mathrel{+}= f(\text{LN}(x))$.
Residual bypasses LN -- clean gradient highway.
Post-LN puts LN on residual, causing gradient instability.
Pre-LN: higher LR safe, no warmup required. GPT-2 onward uses Pre-LN.
\end{interviewbox}

\begin{interviewbox}{Q8: Hardest bug in this project?}
KV-cache off-by-one. Passed seed=42 by coincidence (all 8 tokens were identical value). Failed 9/10 other seeds. Two bugs: wrong first token + positions started at 0.
Found by: \texttt{torch.equal(naive, cached)} for seeds 0,1,2.
\end{interviewbox}

\end{multicols}

\section{Bonus Questions --- Frontier Lab Depth}
%=============================================================================

\begin{multicols}{2}
\footnotesize

\begin{bonusbox}{B1: Prove why $\sqrt{d_k}$ is the right scale}
$q,k\!\sim\!\mathcal{N}(0,1)$. $q{\cdot}k = \sum q_i k_i$.
Each term variance $= E[q_i^2 k_i^2] = 1$.
Sum of $d_k$ iid: $q{\cdot}k \sim \mathcal{N}(0,d_k)$, std $= \sqrt{d_k}$.
Dividing by $\sqrt{d_k}$ normalizes to std$=1$.
\end{bonusbox}

\begin{bonusbox}{B2: RLHF vs this project?}
This: REINFORCE + rule-based reward.
RLHF: PPO + learned reward model from human A-vs-B comparisons.
PPO clips update ratio (stable). Reward model from thousands of labels.
KL structure identical. This is RLHF-lite: same architecture, no human labels.
\end{bonusbox}

\begin{bonusbox}{B3: KV-cache memory at scale?}
$2\!\times\!L\!\times\!B\!\times\!H\!\times\!T\!\times\!d_h\!\times\!\text{bytes}$.
Hypothetical GPT-4 ($L{=}96$, $H{=}96$, $T{=}128$K, $d_h{=}128$, bf16):
$\approx 305$ GB. Solutions: GQA (share KV heads), sliding-window eviction, int8 KV quant, PagedAttention.
\end{bonusbox}

\begin{bonusbox}{B4: Reward hacking and KL tradeoff?}
$\mathcal{L}=-\mathbb{E}[r]+\alpha D_\text{KL}$.
$\alpha{\to}0$: pure RL, degenerate outputs.
$\alpha{\to}\infty$: frozen at SFT.
Rule: larger $\alpha$ when reward is noisy; smaller when trustworthy.
Sweet spot: 0.1--0.3 for demos.
\end{bonusbox}

\begin{bonusbox}{B5: Why does causal mask break in KV-cache?}
Prefill: need lower-triangular mask ($T{\times}T$).
Decode: query is LAST position -- should attend to ALL cached tokens -- no mask needed.
Applying prefill mask to a decode step clips incorrectly.
Fix: skip mask when $T=1$.
\end{bonusbox}

\begin{bonusbox}{B6: Scaling to GPT-3?}
Architecture: GQA, RoPE, bf16.
Training: FSDP (model sharding), activation checkpointing (recompute in backward), grad clip critical.
Inference: Flash Attention (O($T$) memory via tiling), speculative decoding (draft + verify), PagedAttention for batching.
\end{bonusbox}

\begin{bonusbox}{B7: Prove Pre-LN gradient stability}
Post-LN backward: $\partial\mathcal{L}/\partial x$ passes through $\partial\text{LN}/\partial(x+f(x))$.
At depth, $x+f(x)$ has large norm; LN Jacobian $\propto 1/\text{norm}$ destabilizes.
Pre-LN: $\partial\mathcal{L}/\partial x = I + (\ldots)$.
$I$ is gradient highway -- exists regardless of depth. LN input always unit variance.
\end{bonusbox}

\begin{bonusbox}{B8: Why does init loss equal $\ln(50257)$?}
Weights $\sim\mathcal{N}(0,0.02)$: logits$\approx0$. Softmax of zeros = uniform over 50257.
Cross-entropy of uniform: $-\log(1/50257) = \ln(50257) \approx 10.82$.
Measured: 10.8294. Gap $<0.01$ = no pipeline bugs.
\end{bonusbox}

\end{multicols}
